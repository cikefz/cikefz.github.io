上节回顾：

一个伪随机函数PRF，是一个两个输入的函数，取一个密钥和某个集合X里的元素作为输入，输出某个集合Y里的元素，现在唯一的要求是存在一个有效的算法来计算这个函数。

类似地，有一个相关概念叫做伪随机置换，和PRF类似。事实上，也有一个有效算法可以用来计算伪随机置换，但是，伪随机置换多了一个要求，那就是存在一个算法D可以计算函数E的逆。所以一个RPR也是PRF，但函数必须是所有密钥一一映射的，还得有一个有效的求逆算法。

如果攻击者无法区分这两个实验，换句话说，他对实验0输出1的概率跟对实验1输出1的概率几乎一样

Modes of operation : one time key

如何使用一次性密钥的分组密码

electronic code book(电子密码本)

下面介绍第一种安全的构建方法(Deterministic Counter Mode)为了避免ECB的错误加密方式，在处理每一组消息时，我们都会用一个唯一的PRF来进行加密。

Security for many-time key

攻击者可以执行选择明文攻击(Chosen-plaintext attack,CPA)，根据攻击者的需求，他可以自由选择密文和明文组合。攻击者的目标当然都是证明这个加密方案不具备语义安全性质。

solution1 随机加密的算法(Randomized Encryption)

Solution2 基于随机数的加密方案 (Nonce-based Encryption).其中，随机数是会变化的，并且(k,r)永远不会被使用超过一次。

method1是nonce是通过计数器产生的，不用包含在CTs当中，对方知道

Modes of operation: many time key(CBC)

1.密码分组链接模式

L是加密的明文长度，L可以是1000，意味着我们加密的明文，最多有1000个AES分组。q是在CPA攻击下，攻击者获得的密文数，但在现实中，q的意义是我们使用密钥K加密的次数，换句话说，如果我们使用某个AES密钥加密了100条明文信息，q=100这是因为攻击者将会看到用密钥K加密的最多100条信息。

2.

密钥K是用来加密单独的明文分组的，而密钥K1是用来加密非随机但唯一的新鲜值的，这样输出将是一个随机的IV，可以被用在CBC链接中。

许多实例中的错误使用：使用前没有加密IV
